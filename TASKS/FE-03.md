## 🎯 Summary
- 기능명: [FE-03] Confidence Score 80% 미만 구간 0.5초 이내 붉은색 하이라이터 UI 렌더링
- 목적: 파서 엔진이 확신하지 못하는 데이터 노드를 시각적으로 강하게 강조하여, 사용자의 검수(수작업) 시간을 일 4시간에서 30분 이내로 단축하는 북극성 KPI를 달성한다.

## 🔗 References (Spec & Context)
- SRS 문서: 4.1 F2 (REQ-FUNC-007: Confidence 80% 미만 0.5초 내 렌더링)
- SRS 문서: 4.2 Non-Functional Requirements (REQ-NF-002: UI Responsiveness)

## 📋 Task Breakdown
- [ ] `DATA_NODE` 내의 `confidence_score` 값을 평가하여 임계치(80%, 0.8) 미만인 노드를 필터링하는 파생 상태(Derived State) 작성
- [ ] 해당 조건을 만족하는 텍스트/테이블 셀 컴포넌트에 Tailwind CSS 클래스(예: `bg-red-100 border-red-500 text-red-900`)를 조건부로 부여하는 로직 구현
- [ ] 대용량 문서 열람 시에도 하이라이터 렌더링이 지연되지 않도록 CSS 기반의 즉각적인 스타일링 최적화 적용

## ✅ Acceptance Criteria
- **Scenario 1: 저신뢰도 구간 즉각적 시각화**
  - **Given** `confidence_score`가 75%인 데이터 노드와 95%인 데이터 노드가 포함된 문서가 열람될 때
  - **When** 렌더링이 완료되면
  - **Then** 75%인 노드는 붉은색 계열로 확연히 구분되어 강조되고, 95%인 노드는 기본 스타일로 표시되어야 한다.
- **Scenario 2: 렌더링 성능 검증**
  - **Given** 수백 개의 셀이 포함된 결과 화면에서
  - **When** 컴포넌트가 마운트될 때
  - **Then** 저신뢰도 하이라이팅 연산 및 렌더링이 0.5초 이내에 완료되어 사용자가 시각적 지연을 느끼지 않아야 한다.

## 🚧 Constraints
- UI 응답성 (REQ-NF-002): 하이라이터 렌더링은 무조건 0.5초 이내에 완료되어야 하므로 불필요한 React Re-rendering을 최소화해야 한다.
