## 🎯 Summary
- 기능명: [BE-03] 문서 검수 최종 승인 API (`PATCH /api/v1/documents/{doc_id}/validate`)
- 목적: 파싱된 문서의 모든 하이라이트/오류 구간에 대한 인간 검수가 완료되었음을 시스템에 승인(Validate) 처리하여 Export 가능 상태로 전환한다.

## 🔗 References (Spec & Context)
- SRS 문서: 3.3 API Overview (Doc Validate API)
- SRS 문서: 4.1 F2 (REQ-FUNC-009: 문서 전체 검수 승인)

## 📋 Task Breakdown
- [ ] Next.js `PATCH /api/v1/documents/{doc_id}/validate` 엔드포인트 또는 Server Action 구현
- [ ] Prisma를 이용해 `PARSED_DOCUMENT` 테이블에서 `doc_id` 검증
- [ ] 문서의 `validation_passed` 필드를 `true`로, `validated_at`을 현재 시각으로 업데이트
- [ ] 문서 상태가 정상적으로 승인 처리되었음을 프론트엔드에 응답 (상태 코드 200)

## ✅ Acceptance Criteria
- **Scenario 1: 정상적인 문서 검수 승인**
  - **Given** 사용자가 파싱된 문서의 검수를 마치고 승인 버튼을 클릭하면
  - **When** 승인 API가 호출될 때
  - **Then** DB의 `validation_passed`가 `true`로 갱신되고, 이후 해당 문서는 Export 대상에 포함되어야 한다.
- **Scenario 2: 이미 승인된 문서에 대한 중복 처리 방어**
  - **Given** 이미 `validation_passed=true`인 문서에 대해
  - **When** 다시 승인 API가 호출되면
  - **Then** 멱등성(Idempotency)을 보장하여 에러 없이 성공 응답을 주거나, 적절한 무시 처리를 수행해야 한다.

## 🚧 Constraints
- 상태 무결성: 승인 완료된 문서는 이후 `BE-02` (노드 덮어쓰기) 등의 수정을 원칙적으로 차단해야 한다 (UI 및 서버 검증 모두 적용).
