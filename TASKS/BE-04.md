## 🎯 Summary
- 기능명: [BE-04] 승인 문서 대상 데이터 Export API (`GET /api/v1/export`)
- 목적: 최종 검수 승인(`validation_passed=true`)이 완료된 문서의 구조화된 데이터를 JSON 또는 CSV 포맷으로 변환하여 다운로드할 수 있게 제공한다.

## 🔗 References (Spec & Context)
- SRS 문서: 3.3 API Overview (Export API)
- SRS 문서: 4.1 F4 (REQ-FUNC-012, REQ-FUNC-013: Export 포맷 및 조건)

## 📋 Task Breakdown
- [ ] Next.js `GET /api/v1/export` 라우트 핸들러 생성
- [ ] Query Parameter `format` (json, csv) 추출 및 검증
- [ ] Prisma를 통해 `validation_passed=true` 인 `PARSED_DOCUMENT` 및 하위 `DATA_NODE` 조회
- [ ] `format=json` 일 경우 JSON 직렬화 응답 생성 (헤더 `Content-Type: application/json`)
- [ ] `format=csv` 일 경우 CSV 형식으로 데이터 변환 응답 생성 (헤더 `Content-Type: text/csv`)

## ✅ Acceptance Criteria
- **Scenario 1: 승인된 데이터 정상 Export**
  - **Given** `validation_passed=true` 인 문서 데이터가 있을 때
  - **When** 클라이언트가 `GET /api/v1/export?format=csv` 를 요청하면
  - **Then** CSV 파일 형태의 데이터 스트림과 함께 200 상태 코드가 반환되어야 한다.
- **Scenario 2: 미승인 문서 데이터 차단**
  - **Given** `validation_passed=false` 인 문서 데이터가 있을 때
  - **When** Export 데이터 조회 시
  - **Then** 해당 문서의 데이터는 결과물에 절대 포함되지 않아야 한다.

## 🚧 Constraints
- 파일 크기 처리: 다량의 데이터를 Export 할 때 서버 메모리 한계를 넘지 않도록 스트리밍 응답 또는 적절한 버퍼 처리를 고려해야 한다.
- 데이터 정확성: CSV 변환 시 표의 셀 병합이나 계층 구조가 평탄화될 때 의미가 훼손되지 않도록 변환 규칙(옵션)이 일관되어야 한다.
