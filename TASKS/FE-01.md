## 🎯 Summary
- 기능명: [FE-01] 수동 파일 업로드 UI 컴포넌트 및 파싱 상태/오류 코드 안내 창 구현
- 목적: 사용자가 로컬에서 법률 문서를 수동으로 업로드하고, 파싱이 진행되는 동안의 상태(로딩, 성공, 실패 및 오류 코드)를 명확하게 인지할 수 있는 인터페이스를 제공한다.

## 🔗 References (Spec & Context)
- SRS 문서: 3.2 Client Applications
- SRS 문서: 4.1 F1 (손상 문서 판별 시 오류 안내)

## 📋 Task Breakdown
- [ ] `shadcn/ui`를 활용한 Drag & Drop 기반의 파일 업로드 폼(Form) 컴포넌트 구현
- [ ] 업로드 전 클라이언트 단 파일 검증 로직 작성 (지원 형식, 최대 용량 등)
- [ ] BE-01(`POST /api/v1/clean`) 엔드포인트와 연동하여 파일 바이너리 전송 로직 구현
- [ ] 업로드 중 스피너(Spinner) 및 진행률(Progress) 등 파싱 상태(`processing`)를 시각화하는 UI 구현
- [ ] 실패(손상/난독 파일 등) 시 파서에서 반환한 에러 코드를 사용자 친화적 메시지(Toast 또는 Alert)로 표시하는 예외 처리 UI

## ✅ Acceptance Criteria
- **Scenario 1: 정상 업로드 및 파싱 대기 상태 표시**
  - **Given** 사용자가 지원되는 형식의 법률 문서를 업로드했을 때
  - **When** 백엔드로 전송이 시작되면
  - **Then** 화면에 로딩 인디케이터가 표시되고, 파싱 완료 응답이 올 때까지 대기 상태가 유지되어야 한다.
- **Scenario 2: 손상된 파일 업로드 시 피드백**
  - **Given** 파싱이 불가능한 손상된 문서가 업로드되었을 때
  - **When** 엔진으로부터 오류 코드가 반환되면
  - **Then** 로딩 인디케이터가 즉시 해제되고, 화면 중앙 또는 토스트 메시지로 실패 사유(예: "읽을 수 없는 문서입니다")가 안내되어야 한다.

## 🚧 Constraints
- 파일 제한: 사내망 환경을 고려하여 단일 업로드 용량 제한(예: 50MB)을 명시적으로 클라이언트 단에서 1차 차단해야 한다.
