## 🎯 Summary
- 기능명: [FE-02] 엔진에서 반환된 JSON 기반 구조화 데이터 Viewer 렌더링
- 목적: 파서 엔진이 추출한 정형 데이터(`DATA_NODE`)를 사용자가 쉽게 읽고 검수할 수 있도록 화면에 테이블 및 텍스트 형태로 렌더링한다.

## 🔗 References (Spec & Context)
- SRS 문서: 3.2 Client Applications
- SRS 문서: 4.1 F1 (구조화 데이터 추출 확인)

## 📋 Task Breakdown
- [ ] BE-01을 통해 반환된 `DATA_NODE` 배열(JSON 구조)을 프론트엔드 상태(State) 모델로 매핑
- [ ] 노드 타입(`node_type: Table, Text, Form`)에 따라 적절한 UI 컴포넌트로 분기하여 렌더링하는 Document Viewer 레이아웃 구성
- [ ] 병합된 셀(Rowspan, Colspan) 등 복잡한 테이블 구조를 HTML `<table>` 또는 `CSS Grid`를 이용해 정확히 화면에 복원하는 로직 작성
- [ ] 원본 이미지(필요 시)와 추출된 데이터를 Side-by-Side로 대조할 수 있는 레이아웃 스캐폴딩

## ✅ Acceptance Criteria
- **Scenario 1: 복잡한 표 구조 렌더링**
  - **Given** 병합된 셀을 포함하는 JSON 테이블 데이터가 반환되었을 때
  - **When** Viewer 컴포넌트가 렌더링되면
  - **Then** 원본 문서의 표 구조(행, 열 병합 등)가 화면 상에 HTML 표 형태로 어긋남 없이 정확하게 복원되어 표시되어야 한다.
- **Scenario 2: 빈 데이터 또는 잘못된 구조화 데이터 예외 처리**
  - **Given** 파싱 오류로 인해 일부 `DATA_NODE`가 비어 있거나 스키마가 깨진 형태로 전달되었을 때
  - **When** Viewer가 렌더링을 시도하면
  - **Then** 화면 전체가 크래시(White Screen of Death)되지 않고, 해당 노드를 '데이터 없음' 처리하거나 Fallback UI로 안전하게 렌더링해야 한다.

## 🚧 Constraints
- 렌더링 안정성: 수백 개의 셀을 가진 대형 테이블 렌더링 시 브라우저 프리징이 발생하지 않도록 DOM 렌더링 최적화(Virtualization 등)를 고려해야 한다.
