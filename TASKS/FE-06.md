## 🎯 Summary
- 기능명: [FE-06] 승인 완료 데이터 대상 Export (JSON/CSV 선택) 다운로드 버튼 UI 연동
- 목적: 최종 검수가 끝난 정형 데이터를 사용자의 로컬 환경으로 내보내기 위한 다운로드 버튼과 포맷 선택 기능을 구현한다.

## 🔗 References (Spec & Context)
- SRS 문서: 4.1 F4 (REQ-FUNC-012, REQ-FUNC-013: 데이터 Export)
- 연관 API: `GET /api/v1/export` (BE-04)

## 📋 Task Breakdown
- [ ] 문서가 `validated` 상태일 때만 활성화되는 'Export' 버튼 그룹(Dropdown 등) 컴포넌트 생성
- [ ] JSON 및 CSV 포맷을 선택할 수 있는 옵션 메뉴 구현
- [ ] 사용자가 특정 포맷 선택 시 BE-04 API(`?format=json|csv`)를 호출하여 데이터를 Blob 스트림으로 수신
- [ ] 수신된 Blob 데이터를 브라우저 내장 기능(a 태그 download 속성 등)을 통해 파일로 다운로드 처리

## ✅ Acceptance Criteria
- **Scenario 1: 조건부 Export 버튼 활성화**
  - **Given** 문서의 검수 승인 상태(`validation_passed`)에 따라
  - **When** 상태가 `false`일 때는 Export 버튼이 비활성화(Disabled) 처리되고,
  - **Then** 상태가 `true`가 되었을 때만 버튼이 활성화되어 클릭 가능해야 한다.
- **Scenario 2: 파일 정상 다운로드**
  - **Given** 사용자가 Export 버튼에서 'CSV'를 선택하여 클릭했을 때
  - **When** 다운로드 로직이 완료되면
  - **Then** 브라우저에 "문서명_export.csv" 파일이 정상적으로 다운로드 되어야 한다.

## 🚧 Constraints
- 브라우저 호환성: 대용량 Blob 처리 및 파일 다운로드 방식은 주요 모던 브라우저(Chrome, Edge 등)에서 안전하게 동작해야 한다.
