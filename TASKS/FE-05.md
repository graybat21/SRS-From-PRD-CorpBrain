## 🎯 Summary
- 기능명: [FE-05] 덮어쓰기 검수 완료 후 최종 검수 승인 버튼 및 성공 상태 전환 UI
- 목적: 문서 내 모든 검수가 끝난 후 '승인' 액션을 수행하여 문서의 상태를 최종 완료(`validated`)로 전환하는 UI를 제공한다.

## 🔗 References (Spec & Context)
- SRS 문서: 4.1 F2 (REQ-FUNC-009: 전체 검수 승인)
- 연관 API: `PATCH /api/v1/documents/{doc_id}/validate` (BE-03)

## 📋 Task Breakdown
- [ ] 뷰어 하단 또는 상단에 명확하게 인지 가능한 '최종 승인(Validate)' 플로팅 버튼 또는 고정 액션 바 추가
- [ ] 버튼 클릭 시 재확인 모달(Dialog)을 띄워 사용자의 오동작 방지
- [ ] BE-03 API를 호출하여 해당 문서의 검수 완료 상태를 백엔드에 전송
- [ ] 승인 성공 시 화면 내 모든 수정(Input) 기능 비활성화(Read-only) 및 시각적 피드백(예: 초록색 체크 마크, "검수 완료" 배지) 제공

## ✅ Acceptance Criteria
- **Scenario 1: 검수 승인 시 수정 불가능 상태 전환**
  - **Given** 사용자가 문서의 모든 데이터를 검토한 후
  - **When** '최종 승인' 버튼을 클릭하고 모달에서 확인을 누르면
  - **Then** 문서의 상태가 검수 완료로 전환되고, 더 이상 셀 데이터를 수정(Override)할 수 없도록 UI가 잠겨야 한다.
- **Scenario 2: 승인 요청 실패 시 예외 처리**
  - **Given** 네트워크 지연이나 백엔드 에러 상황에서
  - **When** '최종 승인' 버튼을 클릭하면
  - **Then** 문서 상태가 강제로 전환되지 않고, 사용자에게 에러 알림(Toast)이 표시되며, 재시도할 수 있도록 기존 UI 상태가 유지되어야 한다.

## 🚧 Constraints
- 상태 무결성 보장: 승인된 문서에서 UI 조작을 통한 우회 수정이 불가능하도록 철저히 막아야 한다.
