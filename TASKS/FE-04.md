## 🎯 Summary
- 기능명: [FE-04] 하이라이트 된 셀 수동 덮어쓰기(Override) 인풋 박스 및 이벤트 연동
- 목적: 저신뢰도로 표기된 셀의 데이터를 사용자가 직접 수정(Override)할 수 있는 UI를 제공하고, 이를 백엔드(BE-02) API와 연동하여 업데이트한다.

## 🔗 References (Spec & Context)
- SRS 문서: 4.1 F2 (REQ-FUNC-008: 하이라이트 셀 수동 덮어쓰기 기능)
- 연관 API: `PATCH /api/v1/nodes/{node_id}` (BE-02)

## 📋 Task Breakdown
- [ ] 뷰어 내의 셀 클릭 시 텍스트를 수정할 수 있는 Input/Textarea 모드로 전환되는 인라인 편집(Inline Editing) 로직 구현
- [ ] 수정된 값을 입력 후 Enter 키 입력 또는 포커스 아웃(onBlur) 시 변경 사항을 임시 상태에 저장
- [ ] BE-02 API로 `PATCH` 요청을 보내어 DB 상의 데이터를 업데이트하는 HTTP Client 로직 연결
- [ ] 업데이트 요청 중 로딩 상태 표시 및 에러 발생 시 원래 값으로 롤백(Rollback)하는 낙관적 UI(Optimistic UI) 업데이트 처리

## ✅ Acceptance Criteria
- **Scenario 1: 정상적인 데이터 덮어쓰기 및 반영**
  - **Given** 사용자가 하이라이트 된 표의 특정 셀을 클릭했을 때
  - **When** 새로운 값을 입력하고 저장을 유발(엔터/포커스 아웃)하면
  - **Then** 백엔드 API가 호출되어 성공해야 하며, 셀의 내용이 새 값으로 변경되고 하이라이트 스타일이 해제(또는 검수 완료 색상으로 변경)되어야 한다.

## 🚧 Constraints
- 데이터 동기화: 여러 셀을 빠르게 수정할 경우 요청이 유실되지 않도록 디바운싱(Debouncing)이나 큐(Queue) 처리를 고려해야 한다.
